Q&A de HTML Semántico

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de redes, mientras que la World Wide Web es un sistema de información que se encuentra en Internet.

¿Cuáles son las partes de un URL?

Un URL (Uniform Resource Locator) tiene las siguientes partes:

  • Protocolo: HTTP, HTTPS, FTP, etc.
  • Dominio: www.google.com, www.youtube.com, etc.
  • Puerto: 80, 443, etc. (usualmente no se especifica)
  • Ruta: /search, /watch, etc.
  • Query: ?q=hello, ?v=dQw4w9WgXcQ, etc.
  • Fragmento: #section1, #section2, etc.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita una representación de un recurso específico.

HEAD: Solicita una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.

POST: Envía datos al servidor para ser procesados.

PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.

PATCH: Aplica modificaciones parciales a un recurso.

DELETE: Borra un recurso específico del servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST ya que este método envía datos al servidor para ser procesados.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET ya que este método solicita una representación de un recurso específico.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código de respuesta HTTP 200 indica que la solicitud ha tenido éxito. No se ha producido ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. El error 404 indica que el recurso solicitado no se encuentra en el servidor, por lo que el desarrollador debe corregir el sitio web para que el recurso solicitado esté disponible.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. El error 500 indica que ha ocurrido un error interno en el servidor, por lo que el desarrollador debe corregir el sitio web para que el error no se presente.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo HTML5 desaprobado es un atributo que ya no se recomienda utilizar. Algunos elementos de HTML 4 que en HTML5 están desaprobados son:

  • font
  • center
  • strike

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Algunas diferencias principales entre HTML 4 y HTML5 son:

HTML 4 HTML5
Basado en SGML Basado en XML
Requiere el uso de DTD No requiere el uso de DTD
Requiere el uso de etiquetas de cierre No requiere el uso de etiquetas de cierre
Menos semántico Más semántico
Menos soporte para multimedia Más soporte para multimedia
